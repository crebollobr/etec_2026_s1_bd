## 1. O que são funções nativas

Funções já embutidas no MySQL que recebem um valor (ou colunas) e devolvem um resultado **transformado**, dentro do próprio `SELECT`. Não alteram os dados na tabela — só mudam **como** eles aparecem na consulta.

### Ambiente da aula

```sql
CREATE DATABASE nome_sobrenome_20260618;
USE nome_sobrenome_20260618;

CREATE TABLE funcionario (
    id            INT AUTO_INCREMENT PRIMARY KEY,
    nome          VARCHAR(100) NOT NULL,
    email         VARCHAR(120),
    departamento  VARCHAR(40),
    salario       DECIMAL(10,2),
    data_admissao DATE
);

INSERT INTO funcionario (nome, email, departamento, salario, data_admissao) VALUES
('  Ana Silva  ', 'ana@empresa.com',   'TI',     4500.00, '2022-03-15'),
('Bruno Souza',   'BRUNO@EMPRESA.COM', 'Vendas', 2800.50, '2024-07-01'),
('Carla Lima',    NULL,                'TI',     6200.75, '2019-11-30'),
('Diego Alves',   'diego@empresa.com', 'Vendas', 3900.00, '2023-02-10');
```

> Deixamos de propósito: espaços em volta de "Ana", e-mail em maiúsculas, e um e-mail `NULL` — para exercitar as funções.

## 2. Funções de String (texto)

| Função | O que faz |
|--------|-----------|
| `CONCAT(a, b, ...)` | Junta textos |
| `SUBSTRING(txt, ini, tam)` | Recorta parte do texto |
| `UPPER(txt)` / `LOWER(txt)` | Maiúsculas / minúsculas |
| `TRIM(txt)` | Remove espaços das pontas |

```sql
-- Juntar textos
SELECT CONCAT(nome, ' <', email, '>') AS contato FROM funcionario;

-- Recortar: 3 primeiras letras do nome (posição começa em 1)
SELECT nome, SUBSTRING(nome, 1, 3) AS iniciais FROM funcionario;

-- Padronizar caixa
SELECT UPPER(nome) AS maiusculo, LOWER(email) AS email_padrao FROM funcionario;

-- Remover espaços das pontas (compare com e sem TRIM)
SELECT CONCAT('[', nome, ']') AS sem_trim,
       CONCAT('[', TRIM(nome), ']') AS com_trim
FROM   funcionario;
```

> `AS contato` dá um **apelido** à coluna do resultado — deixa a saída mais legível.

## 3. Funções de Data

| Função | O que faz |
|--------|-----------|
| `NOW()` | Data e hora atuais |
| `DATE_FORMAT(data, formato)` | Formata a data como texto |
| `DATEDIFF(data1, data2)` | Diferença em **dias** entre duas datas |

```sql
-- Agora
SELECT NOW() AS agora;

-- Formatar como dd/mm/aaaa
SELECT nome, DATE_FORMAT(data_admissao, '%d/%m/%Y') AS admissao FROM funcionario;

-- Quantos dias de casa cada um tem
SELECT nome, DATEDIFF(NOW(), data_admissao) AS dias_de_empresa FROM funcionario;
```

Códigos úteis do `DATE_FORMAT`: `%d` dia, `%m` mês, `%Y` ano (4 dígitos), `%H:%i` hora:minuto.

## 4. Funções Numéricas

| Função | O que faz |
|--------|-----------|
| `ROUND(num, casas)` | Arredonda |
| `MOD(a, b)` | Resto da divisão |

```sql
-- Arredondar salário para inteiro e para 1 casa
SELECT nome, salario,
       ROUND(salario, 0) AS arredondado,
       ROUND(salario, 1) AS uma_casa
FROM   funcionario;

-- MOD para descobrir par/ímpar (resto da divisão por 2)
SELECT id, MOD(id, 2) AS resto FROM funcionario;
```

## 5. Funções Condicionais

| Função | O que faz |
|--------|-----------|
| `IF(cond, v_verdadeiro, v_falso)` | Decisão simples (2 saídas) |
| `CASE WHEN ... THEN ... END` | Decisão com vários casos |
| `COALESCE(a, b, ...)` | Retorna o primeiro valor **não nulo** |

```sql
-- IF: faixa simples de salário
SELECT nome, salario,
       IF(salario > 5000, 'Alto', 'Normal') AS faixa
FROM   funcionario;

-- CASE: várias faixas
SELECT nome, salario,
       CASE
           WHEN salario >= 6000 THEN 'Sênior'
           WHEN salario >= 3500 THEN 'Pleno'
           ELSE 'Júnior'
       END AS nivel
FROM   funcionario;

-- COALESCE: substitui NULL por um texto padrão
SELECT nome, COALESCE(email, 'sem e-mail cadastrado') AS email FROM funcionario;
```

> `COALESCE` é ideal para tratar colunas que podem vir `NULL` (lembra da Carla sem e-mail).

## 6. Funções de Agregação (resumir vários registros)

As funções anteriores trabalham **linha a linha** (1 linha entra → 1 resultado sai). As de **agregação** são diferentes: pegam **vários registros e devolvem um único valor** — total, média, contagem, mínimo, máximo.

| Função | O que faz |
|--------|-----------|
| `COUNT(*)` | Conta linhas |
| `SUM(col)` | Soma os valores |
| `AVG(col)` | Calcula a média |
| `MIN(col)` / `MAX(col)` | Menor / maior valor |

```sql
-- Quantos funcionários existem
SELECT COUNT(*) AS total_funcionarios FROM funcionario;

-- Soma e média dos salários (a tabela inteira vira UMA linha)
SELECT SUM(salario) AS folha_total,
       AVG(salario) AS media_salarial
FROM   funcionario;

-- Menor e maior salário
SELECT MIN(salario) AS menor, MAX(salario) AS maior FROM funcionario;
```

> Repare: o resultado tem **uma linha só**, mesmo que a tabela tenha milhares de registros.

### `GROUP BY` — agregar por grupo

E para ter o total **por departamento**, em vez do total geral? O `GROUP BY` divide as linhas em grupos e aplica a agregação em **cada grupo**.

```sql
SELECT departamento,
       COUNT(*)     AS qtd,
       SUM(salario) AS folha,
       AVG(salario) AS media
FROM   funcionario
GROUP BY departamento;
```

> Regra prática: tudo que está no `SELECT` e **não** é função de agregação precisa aparecer no `GROUP BY` (aqui, `departamento`).

### `HAVING` — filtrar grupos

O `WHERE` filtra **linhas** (antes de agrupar). Para filtrar **grupos** (depois de agrupar), usa-se `HAVING`:

```sql
-- Só departamentos com mais de 1 funcionário
SELECT departamento, COUNT(*) AS qtd
FROM   funcionario
GROUP BY departamento
HAVING COUNT(*) > 1;
```

## 8. Resumo

| Categoria | Funções |
|-----------|---------|
| String | `CONCAT`, `SUBSTRING`, `UPPER`, `LOWER`, `TRIM` |
| Data | `NOW`, `DATE_FORMAT`, `DATEDIFF` |
| Numérica | `ROUND`, `MOD` |
| Condicional | `IF`, `CASE`, `COALESCE` |
| Agregação | `COUNT`, `SUM`, `AVG`, `MIN`, `MAX` (+ `GROUP BY`, `HAVING`) |

**Boas práticas:**
1. Funções não mudam a tabela — só transformam o resultado do `SELECT`.
2. Use `AS` para nomear as colunas calculadas.
3. Sempre trate `NULL` (com `COALESCE`) quando a coluna pode estar vazia.
4. `CASE` quando há mais de duas saídas; `IF` para decisão simples.
5. **Agregação** resume vários registros num só valor; `GROUP BY` aplica por grupo; `HAVING` filtra grupos (enquanto o `WHERE` filtra linhas, antes de agrupar).
