## 1. Função × Procedure

| | Função (`FUNCTION`) | Procedure (`PROCEDURE`) |
|---|---|---|
| Retorna | **um valor escalar** (`INT`, `VARCHAR`...) | nenhum, **uma linha** ou um **conjunto de resultados** |
| Como usar | dentro de um `SELECT` | com o comando `CALL` |
| Tem `RETURN` | sim, obrigatório | não (usa `SELECT` interno ou parâmetros `OUT`) |

> ⚠️ **Ponto-chave:** no MySQL, `CREATE FUNCTION` **só** retorna um valor escalar. Para devolver uma linha/tupla ou um resultado, o objeto certo é a **procedure**.

### O `DELIMITER`

Funções e procedures têm vários comandos internos separados por `;`. Como o `;` também encerra o comando inteiro, trocamos temporariamente o delimitador (geralmente para `$$`) para o MySQL entender onde o bloco termina:

```sql
DELIMITER $$
-- ... bloco com ; internos ...
END $$
DELIMITER ;   -- volta ao normal
```

## 2. `CREATE FUNCTION` — retorno escalar

Vamos criar uma função que calcula o **salário anual** a partir do mensal.

```sql
DELIMITER $$

CREATE FUNCTION salario_anual(salario_mensal DECIMAL(10,2))
RETURNS DECIMAL(12,2)
DETERMINISTIC
BEGIN
    RETURN salario_mensal * 12;
END $$

DELIMITER ;
```

Anatomia:
- `salario_anual(...)` — nome e **parâmetro de entrada**.
- `RETURNS DECIMAL(12,2)` — o **tipo** do valor devolvido.
- `DETERMINISTIC` — declara que, para a mesma entrada, sempre devolve a mesma saída (o MySQL exige uma característica dessas).
- `RETURN ...` — o valor que sai da função.

Usando a função **dentro de um `SELECT`**, como qualquer função nativa:

```sql
SELECT nome, salario, salario_anual(salario) AS anual FROM funcionario;
```

Outro exemplo, função de texto (classifica o salário):

```sql
DELIMITER $$

CREATE FUNCTION faixa_salarial(valor DECIMAL(10,2))
RETURNS VARCHAR(10)
DETERMINISTIC
BEGIN
    IF valor >= 6000 THEN
        RETURN 'Sênior';
    ELSEIF valor >= 3500 THEN
        RETURN 'Pleno';
    ELSE
        RETURN 'Júnior';
    END IF;
END $$

DELIMITER ;
```

```sql
SELECT nome, salario, faixa_salarial(salario) AS nivel FROM funcionario;
```

## 3. `CREATE PROCEDURE` — retorno de linha/resultado

A procedure é chamada com `CALL` e pode devolver **um conjunto de linhas** (via `SELECT` interno).

### 3.1 Procedure que retorna linhas (`SELECT` interno)

```sql
DELIMITER $$

CREATE PROCEDURE listar_por_cidade(IN p_cidade VARCHAR(60))
BEGIN
    SELECT id, nome, salario
    FROM   funcionario
    WHERE  cidade = p_cidade;
END $$

DELIMITER ;
```

```sql
CALL listar_por_cidade('São Paulo');
```

Retorna **todas as linhas** de funcionários daquela cidade — algo que uma função não consegue fazer.

> `IN` indica parâmetro de **entrada** (o valor que passamos ao chamar).

### 3.2 Procedure com parâmetro `OUT`

Quando queremos devolver **um valor calculado** por uma variável (não uma tabela), usamos `OUT`:

```sql
DELIMITER $$

CREATE PROCEDURE contar_por_cidade(IN p_cidade VARCHAR(60), OUT total INT)
BEGIN
    SELECT COUNT(*) INTO total
    FROM   funcionario
    WHERE  cidade = p_cidade;
END $$

DELIMITER ;
```

```sql
CALL contar_por_cidade('São Paulo', @qtd);
SELECT @qtd AS total_em_sp;
```

- `OUT total` — parâmetro de **saída**.
- `SELECT ... INTO total` — guarda o resultado na variável.
- `@qtd` — variável de sessão que recebe o valor após o `CALL`.

### Gerenciando os objetos criados

```sql
SHOW FUNCTION STATUS WHERE Db = 'nome_sobrenome_20260625';
SHOW PROCEDURE STATUS WHERE Db = 'nome_sobrenome_20260625';

DROP FUNCTION  IF EXISTS salario_anual;
DROP PROCEDURE IF EXISTS listar_por_cidade;
```

## 5. Resumo

| | `FUNCTION` | `PROCEDURE` |
|---|---|---|
| Retorno | 1 valor escalar | linha/resultado ou `OUT` |
| Chamada | dentro de `SELECT` | `CALL` |
| Palavra-chave | `RETURN` | `SELECT`/`OUT` |
| Criar / apagar | `CREATE`/`DROP FUNCTION` | `CREATE`/`DROP PROCEDURE` |

**Boas práticas:**
1. Use `DELIMITER` ao criar o bloco e devolva-o a `;` no fim.
2. Função para **calcular um valor**; procedure para **devolver dados** ou executar uma rotina.
3. Declare `DETERMINISTIC` em funções de cálculo puro.
4. `IN` = entra, `OUT` = sai; variáveis de sessão usam `@`.
